Tier tile density — FB #58

Offline design mockups answering TestFlight feedback #58 (“our tiles are too big”), calibrated against the owner's reference screenshots (KTC-style ~50px single-line rows). All three render the same Tiers-screen fragment — Elite + Starter QB bins, sticky tier banner, FB-90 tier-step chevrons — in a 390×844 iPhone frame with Chalkline tokens and the canonical tier hexes. Compact and cozy also fold in #53 (positional rank prominent) and #54 (0–10k value scale) via the reference app's right-edge “value (QB2)” treatment.

1 · Compact 50px rows

Closest to the reference app: single-line rows with tiny monogram, name + team left, “QB1 · 9,650” right, chevrons collapsed behind a stacked-chevron popover affordance. ~10 rows per screen (vs ~4 today); drops the per-tile tier badge and trend strip.

2 · Cozy 60px rows

Middle ground: two-line rows keeping the TierChalkBadge and a tightened TileStats 30d trend, with explicit side-by-side chevrons (44pt effective — better than today's 40pt). ~8 rows per screen.

3 · Current, tightened −20% ≈92px rows

Comparison honesty: today's exact tile anatomy (badge row, name, meta, stats strip, stacked chevrons) with padding/type reduced ~20% and nothing dropped. Only ~5 rows per screen — the case that #58 needs a re-layout, not just smaller numbers.
