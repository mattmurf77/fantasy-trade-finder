Cozy — 60px two-line rows

  • Row height: 60px (+4px gap = 64px pitch). Roughly 8 player rows per screen vs ~4 today. The middle ground: half today's height without dropping the stat strip.
  • Shown: 3px position rail · line 1 = name + team + injury/RK micro-tags · line 2 = TierChalkBadge kept (preserves #83/#84's per-tile tier signal) + tightened TileStats 30d trend (FB4-61 Consensus|You modes still have a home) · right edge = positional rank prominent (#53) stacked over the 0–10k value (#54).
  • Dropped vs today: PositionChip (redundant with the rail + “QB5”), age/experience meta, headshot slot (no room on a 60px two-line row without crowding the name).
  • FB-90 chevrons: explicit, side-by-side 32×32 icon buttons. Side-by-side matters: stacking two buttons inside a 60px row would leave ~30pt vertical targets (below HIG 44pt); side-by-side each button gets the full row height, and 32×32 visual + hitSlop reaches 44×44pt effective (today's stacked 32px + hitSlop 4 is only 40pt — this variant actually fixes that). Up-chevron shown disabled (45%) on Elite rows — the clamp rule.
  • Tradeoffs: chevron pair costs ~74px of row width, squeezing long names (ellipsis earlier than compact); two text lines at 60px means 11px secondary type — at the floor for comfortable reading.
