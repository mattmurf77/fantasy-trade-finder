Compact — 50px single-line rows

  • Row height: 50px (+4px gap = 54px pitch). Roughly 10 player rows per screen vs ~4 today (~118–122px rows). Closest to the reference app's ~50px rows.
  • Shown: 3px position rail · 22px square monogram (headshot slot) · name + team · RK/injury micro-tags · right edge = positional rank prominent (#53, position-colored mono) with the 0–10k value secondary (#54) — the reference app's “9,815 (QB2)” idea with the prominence inverted per #53.
  • Dropped vs today: PositionChip + TierBadge on the tile (tier now carried only by group headers + sticky banner — a step back from #83/#84's per-tile tier signal), age/experience meta, 30d trend strip (TileStats). Tier group headers gain the reference-app aggregate: count + summed value.
  • FB-90 chevrons: collapsed behind a stacked-chevron affordance (28×36 visual; hit area = full 50px row height × 44pt via hitSlop — meets HIG). Tap opens a tier-step popover with two 44pt actions; shown open on Drake Maye, where Tier up is disabled (clamped at Elite).
  • Tradeoffs: row itself is 50px > 44pt so tap/drag targets are fine, but the extra tap to reach tier-step is a real interaction cost; losing the trend strip removes the FB4-61 stat mode from this screen; circular avatars from the reference are squared to 2px radius (radius rule).
