Variant C — Unified Brief (mode = a lens)

Structure: One continuous canvas holds the whole "trade brief" — direction board, target team, positions, untouchables — always present. The 4-mode selector is a lens: it re-emphasizes the relevant blocks (brightens + auto-scrolls) and dims the rest, rather than swapping screens. Guided pre-fills the brief from roster analysis; Calculator swaps the sticky action into a live fairness verdict. A sticky bottom CTA reflects the active lens.

What's shown

  • Tap a lens (Guided · Team · Player · Calc) — blocks brighten/dim in place; nothing is hidden, so prefs never "disappear."
  • The NBA2K FOR / AWAY board and the position rec-chips live in the same scroll, so cross-referencing is one gesture.
  • Sticky CTA morphs: "Show me trades" (Guided/Team/Player) vs. the live verdict (Calc).

Tradeoffs

  • + Most seamless jumping — mode change is a re-emphasis, zero navigation, all state co-visible.
  • + Doubles as the single prefs home by construction; nothing lives in a separate drawer or screen.
  • − Densest option; the dim-not-hide model can read as busy for a first-timer.
  • − Calculator (a manual builder) is a slightly awkward fit inside a "brief" metaphor — handled by swapping the dock.

Reuse vs. new

  • Reuse: board = FB-47 targeting; positions = OutlookSheet acquire/away; untouchables = asset_prefs #95; verdict = evaluate Mode B (InLeagueCalculator). Lens emphasis is pure client state.
  • New: the unified-canvas shell + lens emphasis engine; "Specific Team" scoping. Recommendation chips: roster needs/surplus already computed server-side (analyze_roster_strengths) but need a client endpoint — only inferred_outlook is exposed today.
