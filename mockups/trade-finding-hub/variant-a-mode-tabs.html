Variant A — Mode Tabs + Shared Prefs Drawer

Structure: A sticky 4-segment mode selector lives directly under the Trades header. The mode body swaps in place; a single Trade Preferences drawer sits below and is shared by every mode — untouchables, positions-needed (with roster-rec chips), positions-to-shed (rec chips), players-for, players-away. Selections persist as you switch modes because they live in one place, not per-mode.

What's shown

  • Tap the mode segments to jump between Fully Guided · Specific Team · Specific Player · Calculator (all clickable).
  • Specific Player uses the NBA2K for / away / both direction toggle feeding a two-column board.
  • The prefs drawer is collapsed by default with a state summary; open it to edit shared prefs.

Tradeoffs

  • + Lowest mental overhead — modes are always visible, one tap to switch, prefs never lost.
  • + Prefs are decoupled from mode, so they're obviously global.
  • − Prefs sit below the fold; a new user may not notice the recommendation chips until they scroll.
  • − Vertical stack gets long in Calculator mode (mode body + drawer + builder).

Reuse vs. new

  • Reuse: mode segments = today's trades.subnav (Trades/Portfolio/Calculator) widened. Specific Player = existing FB-47 finder targeting (pinned_give/receive + PlayerPickerModal). Untouchables = asset_prefs (#95). Positions = OutlookSheet acquire/trade-away chips. Calculator = TradeCalculatorScreen / InLeagueCalculator.
  • New: "Specific Team" mode (pick one league-mate → generate only vs. them). The recommendation chips need a thin endpoint: roster needs/surplus are already computed by analyze_roster_strengths but only inferred_outlook is exposed today — positions rec = data exists, wiring new.
