Near-term vs. modeling

Near-term — buildable on existing data

No new data source. Reskins / recombines what the backend already serves.

  • Value-bar trade verdict /api/trade/evaluate → give_value, receive_value, gap.firsts, gap.pick_equivalent
  • League Summary bar chart (dynasty) /api/league/power-rankings already returns per-position value breakdowns
  • Position filter + live reorder/rescale pure client-side sort over the same payload
  • Roster drill-in, value-sorted by position team.roster is already grouped + value-desc
  • "This is about fit, not value" fit line analyze_roster_strengths → position_needs / position_surplus

Needs new data / modeling — real projects

Each is a standalone build. Don't fake these — flag them.

  • Redraft values FTF has none — backend 501s basis=redraft (dynasty-only source)
  • Season / final-standing projections no weekly-points projection model exists
  • Playoff + championship odds needs standings + remaining schedule + Monte-Carlo sim + playoff-format/bye logic
  • Schedule-aware "+2 wins" deltas re-sim the season with the added/removed player
  • Outlook-tightened 1st-round pick slotting price picks by projected draft slot, not the flat GENERIC_PICK_SEEDS
  • Multi-year (2026/27/28) title odds RosterAudit-style; compounds every dependency above

The two dependencies that gate everything else

1. A redraft / current-season value source. Every "outlook" number starts from projected weekly points. FTF's only value axis today is dynasty (KTC-style, long-horizon). Redraft values, weekly projections, and standings projections are the same missing input viewed three ways. Reference: DynastyDaddy ships both dynasty + redraft; FantasyCalc exposes a redraft toggle.

2. A league-state simulator. Playoff %, title %, and "+2 wins" all require simulating the rest of THIS league's season — standings, remaining schedule, per-matchup win probabilities, and the league's playoff format (seeds, byes). Reference: RosterAudit's multi-year championship odds. This is a projection-model + Monte-Carlo project, not a display change.

Order of operations: (1) redraft/projection values unlock the redraft bar chart AND feed the simulator; (2) the simulator unlocks record/playoff/title deltas; (3) projected draft slots unlock outlook-tightened pick pricing. The value-bar + dynasty bar chart ship independent of all of it.
