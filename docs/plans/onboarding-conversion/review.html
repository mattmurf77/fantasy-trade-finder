Deck-quality gate

AWAITING YOUR SCORING

Offline eval (scripts/deck_eval.py) simulating a brand-new user's first deck across 4 real leagues / 47 teams / 235 cards under exact production flags. Results: empty-deck rate 0.0% (gate <5% ✓), warm-server first deck in ~23ms — the <60s budget is trivially met warm; cold start is the only threat (→ item 3). Insult-rate scoring is yours: the report has blank y/n columns per card — docs/plans/onboarding-conversion/deck-eval-report.md.

Growth P4: trades-first lives or dies on cold-start deck quality. The eval auto-flagged one real engine defect — 2-for-1 consolidations scoring 99% fair while losing ~2,700 consensus value (Daniels+Odunze→Hurts). The fix is running in your spun-off session.

Keep-warm ping

GitHub Actions cronBUILT

A GitHub Actions cron (.github/workflows/keep-warm.yml, every 10 min) pings the prod flags endpoint so Render's free dyno stays awake. Activates when this branch merges to main. Runbook entry added; the ~$7/mo Render Starter memo sits with fin-budget as a recommended-but-non-blocking conversion expense.

04

Trades-first hook screen

onboarding.trades_firstBUILT

First-run Trades is the landing surface: card generation kicks the moment sign-in returns (before the screen even mounts), cards stream into a skeleton deck — the manual "Hit Find a Trade" empty state never shows. Chrome collapses to one row; the invite banner waits until after the first swipe. Three guards on the first impression: the identity strip (wrong-username protection — a typo'd valid username silently loads a stranger's team), the provenance chip (CONSENSUS VALUES → YOUR BOARD), and the visible Bad trade? flag.

Swipes feed Elo at K=8/4 — the user is ranking without knowing it. The scorecard's top interaction (8.9) becomes minute zero of the product.

Username-first landing

onboarding.landingBUILT + ADR-006

The sign-in gate becomes one question. Apple moves off the front door into a quiet re-entry link (P2.6 account-only users have no username to type — without this link they're stranded). Errors are first-class: not-found copy explains username ≠ team name; Sleeper-down elevates the sample-league escape into the error itself.

ADR-006 "account-later": Sleeper auth is a public username read — treating it as signup was self-inflicted friction. Apple keeps its role as the only durable identity; it just asks at the right moment (item 8).

Quick Set prompt + the aha

onboarding.quickset_promptBUILT (incl. backend force-regen)

An inline card takes the top-of-deck slot — first pass after swipe 2, else after 3 swipes, once per session. Accept deep-links into onboarding-mode Quick Set (one position, Quick-Rank offer suppressed) which bounces back to Trades, forces a fresh deck (new backend force param — a board change doesn't alter the job-cache key, so without this the user would see identical cards and the whole loop dies), and shows the receipt banner. Dismiss = snooze → one session-2 re-offer → retires to the provenance chip as the evergreen door. "Dismissed forever" does not exist.

Growth called the regeneration "the most important finding in either round" — the visible deck change is what converts consensus-browsers into board-owners.

Apple save moment + share

onboarding.apple_save_momentonboarding.share_sheetBUILT

The Apple modal fires on first like or first Quick Set save — max one auto-ask per class ever, one per session across classes, honest cross-device framing only (ADR-006 prohibits "save your board"; boards persist without Apple). Decline is one tap, recorded, never immediately re-asked; Settings carries the evergreen link. Session 2+, unbound with ≥5 swipes: one non-modal banner (round-3 ruling — iOS can't ask at backgrounding). Celebrations fire before asks. The share row appears only after a like and never beside the Apple prompt — user-initiated OS share sheet, text + link (card-image share deferred: needs a view-shot dependency).

Guided layer + Assistant GM voice

onboarding.guided_layerBUILT (voice doc + copy applied)

The F2P steal, without the mascot: a swipe-gesture nudge on card 1, ≤4 one-time coach marks (swipe → provenance chip → diff banner → trio entry; never modal, never stacked, all persisted), and two celebration beats timed before the conversion asks. Every string across items 4–10 is the Assistant GM voice — the app works for the user; the user is the GM. The animated mascot is formally deferred: post-launch experiment only if drop-off data says guidance is the leak, and it would need an mkt-brand proposal + Chalkline ADR.
