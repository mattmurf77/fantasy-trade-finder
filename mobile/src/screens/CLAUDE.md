# mobile/src/screens/

One file per top-level route.

| Screen | Purpose |
|---|---|
| `SignInScreen` | Sign in with Apple is the primary portal behind `auth.accounts` (P2.6 account-first: a new Apple identity lands in an account-only session, no league needed); Sleeper username demoted to "Continue with Sleeper" below it (flow unchanged) |
| `LeaguePickerScreen` | Pick which league to use |
| `LeagueScreen` | League home / settings |
| `RankHomeScreen` | Build-your-board chooser ‚Äî Rank tab's first-run screen; describes the five ranking flows by process (guided ‚Üí manual) with Quick set leading as the lowest-effort "recommended" card (#119), saves `rankingMethodPref` so later launches route straight to the chosen flow |
| `RankScreen` | 3-player swipe matchup loop |
| `TiersScreen` | Tiered roster view |
| `QuickSetTiersScreen` | Guided tier quick-set (1.5.4 #104; Rank-stack route `QuickSetTiers`; since #119 a first-class method ‚Äî entered from the Tiers header, the rank-home chooser, the Rank menu, or launch routing via `rankingMethodPref: 'quickset'`) ‚Äî per position, walk the ladder top‚ÜíWaivers (8 steps since #117) tapping player chips into each tier; one `/api/tiers/save` per tier. Finishing offers Quick rank (#136) |
| `QuickRankScreen` | Quick rank (#136; Rank-stack route `QuickRank`) ‚Äî within-tier ordering pass after Quick set: same guided tier-by-tier walk, but tapping stamps click-order rank numbers on the tier's players; Save posts the tier to `/api/rankings/reorder` (clicked order + unclicked appended in current order; subset-safe permutation, tier membership invariant). Skips tiers with <2 players. Entered from the Quick set finish prompt or the Rank menu; NOT a `rankingMethodPref` route (#122 unselected) |
| `PickAnchorScreen` | Pick Anchor wizard (Rank-stack route `Anchors`; reached from the rank home chooser or the Rank menu ‚Äî the Tiers-header link was removed in 1.5.4 #99) ‚Äî value one player at a time in draft-pick terms via `/api/anchor/save`. The #111 pick-value-scale pill row was hidden per #134 (backend `/api/anchor/scale` plumbing + stored scales intact; default 4). Scope pills (#133, PositionTabs construction) filter the queue to All/QB/RB/WR/TE (session-only, module-level mirror); pool query self-heals after the app-resume revalidation race via `useRecoverOnResume` (#121/#125), and the load-error state has pull-to-refresh + Retry (#128) |
| `ManualRanksScreen` | Editable drag/tap rank board ‚Äî labeled "Overall Ranks" in the UI |
| `TradesScreen` | Trade card browser |
| `TradeCalculatorScreen` | Manual trade builder ‚Äî "Calculator" pill in Trades. Live mode: real consensus values via /api/trade/*; Demo mode: mock dual-board league |
| `MatchesScreen` | Mutual trade matches inbox |
| `SleeperConnectScreen` | WebView login to Sleeper ‚Üí captures the JWT for "Send in Sleeper" (flagged beta). Doubles as account **verification** (account-auth P1): the backend proves the captured token live + claim-matched, marks the session verified, and the screen surfaces "Account verified" on success |
| `SettingsScreen` | Settings modal (gear icon; explicit header close control per #130): leagues (+ flag-gated `espn.link` "Link an ESPN league" row ‚Üí LeaguePicker with the sheet auto-opened, #130), ranking pref, notifications, and the **Account** section ‚Äî renders meaningful state for every session type (P2.7): linked identities, an Apple-link button for any session without an Apple identity (binds via `POST /api/auth/apple` on the live session and flips `useSession.verification`), Sleeper source row, link-Sleeper card for account-only users (P2.6 merge flow), verification status, Verify account (Sleeper sessions), Delete account |
| `LeagueSummaryScreen` | League rankings / power rankings (#142/#144; root-stack route `LeagueSummary`, entered from the League tab's "League rankings" Explore row) ‚Äî every team ranked by summed roster value via `GET /api/league/power-rankings`, basis toggle Consensus \| My board (personal values, consensus fallback for unranked players) + a disabled "Redraft (soon)" chip (backend reserves basis=redraft but 501s ‚Äî dynasty values only). Tapping a team opens an overlay of its roster grouped by position, value-sorted within each group, in dense PlayerCard rows |
| `FreeAgentsScreen` | Free-agent finder (#143; root-stack route `FreeAgents`, entered from the League tab's "Free agents" row) ‚Äî best available players in the league ranked by the caller's board values (consensus fallback per unranked player; `user_has_rankings:false` shows a "consensus values" notice), QB/RB/WR/TE filter pills, dense PlayerCard rows with value + FA positional rank, and a "Drop: <player> (+delta)" subline when the backend found a lower-valued same-position roster player to cut. Backend `GET /api/league/free-agents` |
| `PlaceholderScreen` | Stub for unfinished routes |
