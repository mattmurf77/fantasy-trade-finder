### Objective
Exercise the superflex branch end-to-end (the one major engine path the suite
hadn't covered) and prove the two formats are fully isolated rank sets.

### Scope
- **In scope:** sf_tep trio→rank3→generate via the header; format-partitioned
  persistence; **1qb_ppr isolation** (sf_tep ranking must not touch it);
  per-format independent Elo for the same player; sf_tep card validity.
- **Out of scope:** superflex-specific QB-premium *magnitude* tuning; tier-band
  differences per format.

### Actual Result
**8/8 PASS.** 3 sf_tep trios → +9 `swipe_decisions` rows (3×3 pairwise) all
tagged `sf_tep`; **1qb_ppr rank rows unchanged (222→222)** — clean isolation.
sf_tep `member_rankings` partition created (0→685). sf_tep generation → 31 valid
cards (roster-owned, fairness ∈ [0,1]). **Same player held 1qb=1605 vs sf=1800
Elo** — a QB correctly valued higher in superflex. Evidence:
`qa/e2e/scratch/TC-E2E-003-run.json`.

### Outcome
**PASS** — the superflex path works end-to-end and the dual-format partition is
genuinely independent; ranking in one format cannot corrupt the other.

### Observations & feedback (no change required)
- **The 1qb≠sf Elo divergence (1605 vs 1800) is the format working as intended**
  — superflex inflates QB value, and the independent rank sets capture it. Good
  real-data confirmation that the value space isn't shared across formats.
- **Format is header-scoped per call** (`X-Scoring-Format`) *or* session-sticky
  (`/api/scoring/switch`). The header path is what mobile uses for transient
  format views — verified it routes to the right service without a state change.
- sf_tep `member_rankings` auto-publish (685 rows) fires on the first sf_tep
  rank3, same as 1qb_ppr — so leaguemates immediately see the user's superflex
  valuations. Consistent dual-write behavior across formats.
