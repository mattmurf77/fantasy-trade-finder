### Objective
Cover the multi-league surfaces and specifically the **cross-league
disposition** branch — accepting a match that belongs to a league other than the
session's active one (which skips the in-memory Elo apply and relies on
persistence to replay into the correct league's service on its next init).

### Scope
- **In scope:** matches/all spanning leagues; awaiting array contract; portfolio
  over multiple `league_ids`; create a fresh match in league A, switch session to
  league B, disposition the A match — verify 200, decision persisted, Elo signal
  queued for replay.
- **Out of scope:** the actual replay-into-A's-service on next init (verified
  indirectly via the persisted signal); portfolio valuation correctness.

### Observations & feedback (no change required)
- **The match fired on whichever swipe completed the mirror** — because the
  counterparty had a pre-existing like in the fixture, the *partner's* swipe
  created the match, not the mirror swipe. A reminder that match creation is
  symmetric (either side's like can close it) and that tests must locate the
  match by DB state, not by which response carried the id. Reinforces the
  TC-E2E-001 F-2 note about unbounded match dedup interacting with prior likes.
- **Cross-league disposition is correctly league-scoped**: the match carries its
  own `league_id`, so the caller's active session league is irrelevant to where
  the decision lands — the cross-league Matches inbox can accept/decline from any
  league without a context switch. Verified the persisted decision targeted the
  match's league, not the session's.
- **The Elo-replay-via-persistence design holds**: rather than mutate the wrong
  league's in-memory service, the signal is written to `swipe_decisions` and
  replays on that league's next session_init — the same durability that makes
  TC-E2E-002 (restart resilience) work.
