# In-progress handoff — Tiers drag coord-space fix

Last updated: 2026-05-21 (around build #8 submission)

## What's in flight right now

A subagent is fixing the Tiers screen drop coordinate-space bug. Triggering
feedback: row #23 in prod `app_feedback`:

> Tier drag and drop is completely broken. Tiles don't "move" as you're
> dragging and the "drop" location is way off. The tile is dropping
> outside of the field view, putting players much lower than intended.

Diagnosis: `setBinLayout` / `setChipLayout` in
`mobile/src/screens/TiersScreen.tsx` capture `e.nativeEvent.layout.y`
which is **relative to the parent ScrollView's content view** (small
numbers). The gesture passes `e.absoluteY` (screen coordinates,
including header + safe area, larger numbers by ~80px). `zoneAt` /
`dropTargetAt` compare the two, so drops land one or two bins too
low.

Subagent task brief: convert bin + chip layouts to screen-Y via
`measureInWindow`, push to a branch named `fix/tiers-drop-coord-space`.
Should NOT open the PR itself.
